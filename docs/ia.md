# [Use Case] HobbyFind: 취미 탐색 서비스

## 1. 액터 정의 (Actor Definitions)

| 액터명 | 정의 | 역할 |
| :--- | :--- | :--- |
| **비로그인 사용자 (User)** | 별도의 가입 없이 서비스를 이용하는 모든 방문객 | 전체 취미 탐색, 카테고리별 필터링 기능 이용 |
| **시스템 (System)** | HobbyFind 웹 서비스 | 취미 데이터 제공, 카테고리별 페이지 라우팅, UI 렌더링 |

---

## 2. 상세 Use Case 시나리오 및 이벤트 흐름

### **UC-01. 루트 페이지 전체 취미 탐색**
* **개요**: 사용자가 첫 화면에서 18종의 전체 취미 리스트를 확인하는 과정
* **사전 조건**: 사용자가 HobbyFind URL로 접속함
* **사후 조건**: 전체 취미 목록이 그리드 형태로 화면에 표시됨

| 단계 | 액터 (Actor) | 시스템 (System) |
| :--- | :--- | :--- |
| 1 | 루트 페이지(`/`)에 접속함 | 서비스 로고, Top Bar, Hero 섹션을 렌더링함 |
| 2 | 메인 콘텐츠 영역으로 스크롤함 | 고정된 18종의 취미 데이터를 그리드 형식으로 노출함 |
| 3 | 개별 취미 카드를 확인함 | 각 카드의 취미명과 카테고리 태그를 시각화함 |

### **UC-02. 카테고리별 취미 필터링 탐색**
* **개요**: 특정 성향(운동/지능/예술)에 맞는 취미만 선별하여 확인하는 과정
* **사전 조건**: 사용자가 루트 페이지 혹은 특정 카테고리 페이지에 위치함
* **사후 조건**: 선택된 카테고리에 해당하는 6종의 취미만 표시됨

| 단계 | 액터 (Actor) | 시스템 (System) |
| :--- | :--- | :--- |
| 1 | Top Bar의 카테고리 메뉴(운동/지능/예술 중 택 1)를 클릭함 | 클릭된 카테고리의 전용 경로(예: `/category/exercise`)로 이동함 |
| 2 | 페이지 상단 카테고리 정보를 확인함 | 선택된 카테고리 명칭과 해당 카테고리 전용 소개 문구를 표시함 |
| 3 | 필터링된 그리드 리스트를 탐색함 | 전체 데이터 중 해당 카테고리에 속한 6개의 취미 카드만 렌더링함 |

---

## 3. 대체 흐름 및 엣지 케이스 (Alternative & Edge Cases)

* **대체 흐름 (로고 클릭)**: 사용자가 카테고리 페이지 탐색 중 Top Bar의 로고를 클릭하면 즉시 루트 페이지(`/`)로 복귀하며 전체 리스트를 다시 노출한다.
* **엣지 케이스 (잘못된 URL 접근)**: 사용자가 정의되지 않은 카테고리 URL(예: `/category/unknown`)로 강제 접근할 경우, 404 페이지를 노출하거나 루트 페이지로 리다이렉트 처리한다.

---

## 4. 비즈니스 규칙 및 제약사항 (Business Rules)

1.  **데이터 고정**: 취미 목록은 제공된 18개(운동 6, 지능 6, 예술 6)로 고정하며, 사용자나 관리자가 임의로 추가할 수 없다.
2.  **이용 제한 없음**: 모든 페이지와 기능은 로그인을 요구하지 않으며, 세션 정보나 쿠키를 저장하지 않는다.
3.  **기능 범위**: 단순 탐색 서비스이므로 '좋아요', '상세보기', '공유하기' 등의 부가 기능은 일체 구현하지 않는다.

---

## 5. UI 및 데이터 요구사항

### **5.1. UI 고려사항**
* **일관성**: Top Bar는 모든 페이지(루트, 카테고리) 상단에 고정되어야 한다.
* **반응성**: 모바일 환경에서도 취미 그리드 카드가 깨지지 않도록 반응형 레이아웃(Grid)을 적용한다.
* **시각적 구분**: 카테고리별로 카드 태그의 색상을 다르게 적용하여 직관성을 높인다.

### **5.2. 데이터 요구사항 및 흐름**
* **Data Flow**: `Static Data Object` → `Root Page (All)` / `Category Page (Filter by Category Prop)`
* **필수 데이터 필드**: `id`, `name`, `category`, `description` (소개 문구용)

---

## 6. 보안 및 예외 처리

* **보안**: 개인정보를 수집하지 않으므로 개인정보 보호 이슈는 없으나, HTTPS 적용을 통해 서비스 신뢰성을 확보한다.
* **예외 처리**:
    * 네트워크 오류로 데이터를 불러오지 못할 경우, "취미 정보를 불러올 수 없습니다"라는 안내 메시지를 그리드 영역에 표시한다.
    * 비정상적인 경로 접근 시 Next.js의 `notFound()` 기능을 활용하여 표준 안내 페이지를 제공한다.
