# [PRD] HobbyFind: 취미 탐색 및 분류 웹서비스

## 1. 제품 상세 설명 (Detailed Product Description)
* **서비스명**: HobbyFind
* **핵심 목표**: 사용자가 복잡한 절차 없이 자신의 성향(운동형, 지능형, 예술형)에 맞는 취미를 직관적으로 탐색하고 선택할 수 있도록 돕는 큐레이션 웹 플랫폼입니다.
* **가치 제안**: 
    * **접근성**: 비로그인 기반으로 누구나 즉시 이용 가능합니다.
    * **명확성**: 고정된 18개의 취미를 3대 카테고리로 분류하여 정보 과부하를 방지합니다.
    * **직관성**: 그리드 형태의 UI를 통해 한눈에 취미 리스트를 파악할 수 있습니다.

## 3. 핵심 기능 및 사양 (Core Features and Specifications)

| 구분 | 구성 요소 | 상세 기능 설명 |

| | **Hero 섹션** | - '당신에게 맞는 취미를 찾아보세요' 등 서비스 정체성을 나타내는 텍스트 배치 |

## 4. 사용자 페르소나 및 시나리오 (User Persona and Scenarios)

### 4.1. 페르소나
* **이름**: 김취미 (28세, 직장인)
* **상황**: 퇴근 후 생산적인 활동을 하고 싶지만 무엇을 시작해야 할지 모름. 복잡한 회원가입이나 앱 설치를 꺼려함.

### 4.2. 사용자 시나리오
1.  **접속**: 브라우저를 통해 `HobbyFind` 루트 페이지에 접속합니다.
2.  **탐색**: 메인 그리드에서 전체적인 취미 리스트를 훑어봅니다.
3.  **필터링**: 평소 몸을 쓰는 활동에 관심이 있어 Top Bar의 **'운동형'** 필터를 클릭합니다.
4.  **선택**: 운동형 전용 페이지에서 '클라이밍'과 '요가'를 확인하고 자신의 최종 취미 후보로 결정합니다.

## 5. 기술 스택 추천 (Technical Stack Recommendations)

* **Framework**: **Next.js (App Router)**
    * 사유: SEO 최적화 및 카테고리별 동적 라우팅(Dynamic Routing) 구현에 용이함.
* **Styling**: **Tailwind CSS**
    * 사유: 그리드 레이아웃(Grid) 및 반응형 UI를 빠르게 구현 가능함.
* **Language**: **TypeScript**
    * 사유: 취미 데이터 객체의 타입을 정의하여 개발 시 안정성 확보.
* **Deployment**: **Vercel**
    * 사유: Next.js와의 최적화된 호환성 및 빠른 배포 환경 제공.

---

## 6. 제한사항 (Limitations)

* **기능 제한**: 검색(Search) 기능, 회원가입/로그인, 마이페이지, 취미 상세 정보 페이지는 구현 범위에서 제외한다.
* **데이터 제한**: 제공된 18개 취미 목록 외에 데이터를 임의로 추가하거나 수정하지 않는다.
* **페이지 제한**: 루트 페이지와 3개의 카테고리별 전용 페이지(총 4개 타입) 외의 추가 페이지는 구성하지 않는다.
